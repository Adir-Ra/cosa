Frequently Asked Questions

The following are answers to some of the most common questions or concerns that have been raised to COSA.

Who

  • Who can have a project membership in COSA?
      Anyone who has an open-source project focused on implementing or developing CDISC standards to drive innovation in the CDISC community and meet the COSA evaluation criteria.
  • Who decides if a project can be a member of COSA?
      The COSA board evaluates each applicaiton for mmebership in COSA against published membership evaluation criteria.
  • Who manages the open-source project once it has membership in COSA?
      Each open-source project that applies for membership, must provide a project governance board to manage the project responsibilities. The COSA board provides oversight, and acts as a remediation body for challenges project boards may encounter.
What

  • What is COSA?
      COSA stands for the CDISC Open Source Alliance. CDISC saw the need for providing a platform that would enable open-source developers to highlight CDISC standards projects to drive industry innovation.
  • What open-source projects can be a member of COSA?
      Any open-source project that adheres to the COSA charter and evaluation criteria may request membership in COSA. This could be open-source projets that are started in COSA, open-source projects that COSA links to, or open-source projeects that are moved into COSA.
  • What license must an open-source project have to be part of COSA?

Where

  • Does my COSA approved open-source project need to reside in CDISC's GItHub?
      No, COSA is an index of projects that have been evaluated and found to implement or develop CDISC standards to drive innovation in the CDISC community. These project can be in managed in the CDISC GitHub space, in a project's own GitHub space, and even in another companies' GitHub space.
  • Where can I go to place a request open-source project be included with COSA?
      Clicking on "Request COSA Membership" will take you to a page with the COSA request form as well as criteria upon which the request will be evaluated. Complete the form and email to the address found at the top of the form.

Why

  • Why is CDISC creating COSA?
      CDISC has increasingly been approached by members of the community requesting the support and promotion of open-source projects that are focused on driving innovation with CDISC standards.
  • Why would I want to host my eligible project with COSA?
      Hosting your eligible project with COSA, provides you with avenues of communication (GitHub, CDISC Website, Azure, Social Media, Email), events (Interchange, Hackathons/Connectathon, Webinars, etc.) and numerous collaboration opportunities.
  • Why might my project not be accepted by COSA?
      After scrutinizing the project against COSA charter and evaluation criteria, an open-source project may be denied membership if it cannot be proven to be focused on implementing or developing CDISC standards to drive innovation in the CDISC community.

When

  • I submitted my project to COSA; when will I hear if it's been accepted?
      The COSA board minimally meets once per week. If the request provides enough information to allow the board to decide if the project is eligible, then it will be compared with the evaluation criteria and voted upon for inclusion. If there are questions pertaining to the project, the board may reach out to the requester to obtain more information before a final vote is made
